# 研究与 Notebooks 地图

如果你想知道这个仓库后期真正做了什么，优先看 `notebooks/`，不要优先看旧教程。

## 总体判断

这些 notebook 基本组成了一条很清晰的研究链，而且明显参考了 *Advances in Financial Machine Learning*：

- event-driven bars
- fractional differentiation
- CUSUM filter
- triple barrier labeling

也就是说，后期重点已经从“写一个完整框架”转向“围绕数据处理、标签和研究方法做实验”。

## 每个 notebook 在做什么

### `notebooks/data_process/crypto_data_download.ipynb`

- 调用 `RawDataDownloader`
- 下载 Binance 的 `aggTrades` 或 `klines`
- 是整个研究链的数据入口

### `notebooks/data_process/generate_bar.ipynb`

- 调用 `BarGenerator` 和 `BarRule`
- 生成 Dollar / Volume / Tick Bars
- 核心关注点是 event-driven sampling，不是固定时间 K 线

### `notebooks/data_process/handle_price.ipynb`

- 研究 return 分布
- 做 ADF 检验
- 搜索并计算 FFD（分数阶差分）
- 目标是尽量保留记忆的同时把序列处理得更平稳

### `notebooks/data_process/add_label.ipynb`

- 先做动态波动率
- 再做 CUSUM filter
- 最后做 Triple Barrier Labeling
- 这是最接近 AFML 标准监督学习样本构造流程的 notebook

### `notebooks/research/analyze_dollar_bar_stats.ipynb`

- 分析生成后的 dollar bar 数量
- 看日内 bar 数、时间间隔、成交额分布
- 更偏研究 bar 的统计性质

### `notebooks/research/analyze_dollar_bar_threshold.ipynb`

- 比较静态阈值和动态阈值
- 看不同目标 bar 数下的稳定性
- 本质上是在研究“怎样让 sampling 频率更稳定”

## 这些 notebook 和代码目录怎么对应

- 数据下载：`crypto_data_download.ipynb` <-> `src/data_handler/raw_data_downloader.py`
- 生成 bars：`generate_bar.ipynb` <-> `src/data_handler/bar_generator.py`
- 统计和平稳性：`handle_price.ipynb` <-> `docs/notes/data_processing.md`
- 标签：`add_label.ipynb` 是当前最重要的监督学习样本构造说明

## 还要不要看 `src/strategies/`

可以看，但要知道它不是当前主线。

- [`src/strategies/RSIStrategy.py`](../../src/strategies/RSIStrategy.py)
  是一套较早的向量化策略接口
- 它和 `src/backtest/strategies.py` 是两条不同思路
- 现在更像历史原型，而不是 notebook 研究链的中心

## 建议阅读顺序

1. `crypto_data_download.ipynb`
2. `generate_bar.ipynb`
3. `handle_price.ipynb`
4. `add_label.ipynb`
5. 两个 `research/` notebook

这五步看完，基本就能理解这个仓库后期的研究方向。
