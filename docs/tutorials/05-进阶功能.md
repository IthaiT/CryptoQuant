# 项目现状与遗留文件

这份清单不是“垃圾文件列表”，而是按当前可用性给仓库做一次现实分层。

## A. 当前主线

- [`app.py`](../../app.py)
  当前最真实、最完整、最直接可用的系统
- [`saved_params.json`](../../saved_params.json)
  给 `app.py` 持久化参数
- `notebooks/`
  后期主要研究成果

## B. 仍有价值，但不是主入口

- [`src/data_handler/`](../../src/data_handler)
  数据下载和 bar 生成，仍然是有用资产
- [`src/backtest/`](../../src/backtest)
  仍然能看出完整回测框架，但需要继续修整
- [`script/analyze_normality.py`](../../script/analyze_normality.py)
  独立统计分析脚本，可作为研究辅助工具
- [`script/merge_klines.py`](../../script/merge_klines.py)
  合并 K 线有用，但和回测 loader 目前不完全兼容
- [`script/plot_investment_formula.py`](../../script/plot_investment_formula.py)
  和财务规划逻辑相关，更像公式验证/画图脚本，不是系统核心

## C. 明显已经脱节，建议视为遗留文件

### 1. `tests/test_data_handler.py`

问题很明确：

- 还在从 `src.data_handler.bar_generator` 导入 `DollarBar` / `VolumeBar` / `TickBar`
- 当前代码里并没有这些类
- 说明测试对应的是更早的一版 API

结论：这不是当前代码的有效测试。

### 2. `tests/test_strategy_return.py`

问题也很明确：

- 它假设 `RSIStrategy.run()` 的参数与行为是另一套旧接口
- 它只覆盖 `src/strategies/`，不覆盖当前 `src/backtest/strategies.py`

结论：这是旧策略接口的历史测试，不是当前系统验收测试。

### 3. `script/simulation_plate.py`

特点：

- 只被旧 README 提到
- 依赖 Binance Testnet 和 `.env`
- 走的是 `src/strategies/RSIStrategy.py` 那套早期接口

结论：更像早期拷贝/实验脚本，不应被当成当前主功能。

## D. 不是系统功能，而是知识库/文档工具

- [`docs/notes/`](../../docs/notes)
  个人量化知识库、思维导图、研究笔记
- [`script/km_to_md.py`](../../script/km_to_md.py)
  把 `.km` 思维导图转成 Markdown
- [`script/export_quant_map_to_obsidian.py`](../../script/export_quant_map_to_obsidian.py)
  把量化路线图导出到 Obsidian

这些文件不等于“没用”，但它们不属于项目主程序。

## E. 当前最重要的代码漂移

### 1. 仓库名字和主功能已经不一致

`CryptoQuant` 这个名字会让人误以为主入口是量化回测，但现在主入口其实是财务规划器。

### 2. `pyproject.toml` 没覆盖 `app.py` 依赖

`app.py` 用到的 `streamlit`、`plotly` 不在项目依赖里。

### 3. 下载链路和回测链路没有完全接上

- 下载/合并脚本输出一套列名
- 回测 loader 期待另一套旧列名

## 如果你后面想清理仓库

最合理的做法不是直接删，而是先分目录：

- `app/` 或 `finance_app/`
- `quant_research/`
- `legacy/`
- `knowledge_base/`

这样以后看仓库就不会再混淆“当前系统”和“历史资产”。
