# 项目现状与遗留文件

这份清单不是“垃圾文件列表”，而是按当前可用性给仓库做一次现实分层。

## A. 当前主线

- [`wealth_planner/app.py`](../../wealth_planner/app.py)
  当前最真实、最完整、最直接可用的系统
- [`wealth_planner/saved_params.json`](../../wealth_planner/saved_params.json)
  给 `wealth_planner/app.py` 持久化参数
- `notebooks/`
  后期主要研究成果

## B. 仍有价值，但不是主入口

- [`src/data_handler/`](../../src/data_handler)
  数据下载和 bar 生成，仍然是有用资产
- [`src/backtest/`](../../src/backtest)
  仍然能看出完整回测框架，但需要继续修整
- [`script/analyze_normality.py`](../../script/analyze_normality.py)
  独立统计分析脚本，可作为研究辅助工具
- [`script/merge_klines.py`](../../script/merge_klines.py)
  合并 K 线有用，但和回测 loader 目前不完全兼容
- [`wealth_planner/scripts/plot_investment_formula.py`](../../wealth_planner/scripts/plot_investment_formula.py)
  和财务规划逻辑相关，更像公式验证/画图脚本，不是系统核心

## C. 测试现状

### 1. 当前保留的测试

现在的自动化测试已经重建为 3 个 `pytest` 文件：

- [`tests/test_bar_rule.py`](../../tests/test_bar_rule.py)
- [`tests/test_bar_generator.py`](../../tests/test_bar_generator.py)
- [`tests/test_raw_data_downloader.py`](../../tests/test_raw_data_downloader.py)

当前覆盖范围很克制，只验证 `src/data_handler/` 这条主数据链：

- `BarRule` 的累计与关 bar 逻辑
- `BarGenerator` 的跨天聚合和分 period 生成
- `RawDataDownloader` 的列标准化与压缩写盘

运行方式：

```bash
pip install -e ".[dev]"
pytest -q
```

### 2. 已删除的旧测试

- `tests/test_data_handler.py`
- `tests/test_logger.py`

它们对应的是更早的 API 和“脚本式验证”，已经不再代表当前代码状态。

## D. 不是系统功能，而是知识库/文档工具

- [`docs/notes/`](../../docs/notes)
  个人量化知识库、思维导图、研究笔记
- [`script/km_to_md.py`](../../script/km_to_md.py)
  把 `.km` 思维导图转成 Markdown

  把量化路线图导出到 Obsidian

这些文件不等于“没用”，但它们不属于项目主程序。

## E. 当前最重要的代码漂移

### 1. 仓库名字和主功能已经不一致

`CryptoQuant` 这个名字会让人误以为主入口是量化回测，但现在主入口其实是财务规划器。

### 2. `wealth_planner/app.py` 仍有可选依赖

`wealth_planner/app.py` 的核心依赖已经写进 `pyproject.toml`，但敏感性分析相关的 `scikit-image` 仍然放在可选依赖里。

### 3. 下载链路和回测链路没有完全接上

- 下载/合并脚本输出一套列名
- 回测 loader 期待另一套旧列名

### 4. 一条已清理的死链

- 旧的 `src/strategies/` 向量化策略接口已删除
- 依赖它的 `tests/test_strategy_return.py` 和 `script/simulation_plate.py` 也已删除

## 如果你后面想清理仓库

最合理的做法不是直接删，而是先分目录：

- `app/` 或 `finance_app/`
- `quant_research/`
- `legacy/`
- `knowledge_base/`

这样以后看仓库就不会再混淆“当前系统”和“历史资产”。
