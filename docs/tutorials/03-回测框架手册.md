# 回测框架手册

这套回测代码仍然能说明项目早期架构，但它不是当前主入口，也存在明显代码漂移。

## 关键文件

- [`src/backtest/engine.py`](../../src/backtest/engine.py)
  `BacktestEngine` 和 `quick_backtest`
- [`src/backtest/data_loader.py`](../../src/backtest/data_loader.py)
  把 CSV 转成 Backtrader `PandasData`
- [`src/backtest/strategies.py`](../../src/backtest/strategies.py)
  `RSIBacktraderStrategy` 和 `MABacktraderStrategy`
- [`src/backtest/analyzers.py`](../../src/backtest/analyzers.py)
  交易分析器和手续费模型
- [`src/backtest/realtime_chart.py`](../../src/backtest/realtime_chart.py)
  本地 FastAPI + Lightweight Charts 实时图表
- [`src/backtest/visualizer.py`](../../src/backtest/visualizer.py)
  静态 PNG/TXT 报告

## 它实际上做什么

回测流程是：

1. 读取一份已经整理好的 K 线 CSV
2. 用 Backtrader 执行策略
3. 本地起一个 Web 图表服务显示 K 线和交易标记
4. 回测结束后输出统计结果和静态报告

## 当前自带策略

- `RSIBacktraderStrategy`
  一个简单的 long-only RSI 策略
- `MABacktraderStrategy`
  一个双均线示例

这里的策略接口属于 `src/backtest/strategies.py`，和 `src/strategies/` 那套较早的向量化接口不是一回事。

## 启动脚本

入口是：

- [`script/run_backtest.py`](../../script/run_backtest.py)

但它有两个明显的历史痕迹：

- 还在兼容 `Data/` 和 `data/` 两种目录名
- 提示里提到的 `Data/BTCUSDT_data_download.py` 在仓库里并不存在

## 现在最需要知道的限制

### 1. 数据格式不一致

[`src/backtest/data_loader.py`](../../src/backtest/data_loader.py) 期望的是旧式列名：

- `Open time`
- `Open price`
- `High price`
- `Low price`
- `Close price`
- `Volume`

而当前下载/合并脚本产出的列名通常是：

- `open_time`
- `open`
- `high`
- `low`
- `close`
- `volume`

所以旧回测线并不是“下载完就能直接跑”。

### 2. 文档和测试长期没同步

原来的教程、README 和测试文件大量引用了已经变化的 API。

### 3. 更适合当作可维护资产，而不是成品

这套代码的价值主要在于：

- 看架构
- 继续改造
- 为 notebooks 的研究结果找落地框架

而不是作为当前仓库最稳定的用户入口。
