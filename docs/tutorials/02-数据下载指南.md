# 数据下载指南

这条线服务的是早期加密研究，不是 `app.py`。

## 数据链路

最核心的路径是：

1. 下载 Binance 原始数据
2. 生成 event-driven bars
3. 做统计分析、标签或回测输入

对应代码：

- [`src/data_handler/raw_data_downloader.py`](../../src/data_handler/raw_data_downloader.py)
- [`src/data_handler/bar_generator.py`](../../src/data_handler/bar_generator.py)
- [`src/data_handler/bar_rule.py`](../../src/data_handler/bar_rule.py)

## 1. 下载原始数据

`RawDataDownloader` 支持两类数据：

- `aggTrades`: 聚合成交明细，适合生成 Dollar/Volume/Tick Bars
- `klines`: 标准 OHLCV K 线

最小示例：

```python
from src.data_handler.raw_data_downloader import RawDataDownloader

downloader = RawDataDownloader(
    data_type="aggTrades",
    symbol="BTCUSDT",
    start_date="2025-01-01",
    end_date="2025-01-07",
)
downloader.download()
```

默认目录：

```text
data/raw_data/
└── BTCUSDT/
    ├── aggTrades/YYYY/MM/*.csv.zst
    └── klines/1m/YYYY/MM/*.csv.zst
```

特点：

- 从 Binance Vision 拉公开数据
- 每日文件落地为 `.csv.zst`
- 支持多线程下载

## 2. 生成 bars

`BarGenerator` 读取的是 `aggTrades`，不是 `klines`。

内置规则：

- `DollarBarRule`
- `VolumeBarRule`
- `TickBarRule`

最小示例：

```python
from src.data_handler.bar_generator import BarGenerator
from src.data_handler.bar_rule import DollarBarRule

gen = BarGenerator()
rule = DollarBarRule(threshold=500_000)
df = gen.generate(rule, symbol="BTCUSDT", start_date="2025-01-01", end_date="2025-01-07")
```

这个实现有两个值得注意的点：

- 支持跨天连续聚合，不会在自然日强制截断
- `DollarBarRule` 里实现了动态阈值更新思路，而不只是固定阈值

## 3. K 线合并

[`script/merge_klines.py`](../../script/merge_klines.py) 可以把每日 `klines` 合并成一个 CSV。

但要注意一个现实问题：

- `merge_klines.py` 输出的是 `open_time/open/high/...`
- [`src/backtest/data_loader.py`](../../src/backtest/data_loader.py) 期待的是旧格式 `Open time/Open price/...`

也就是说：

- “下载 -> 合并 -> 回测” 这条链路现在不是完全打通的
- 如果你要继续用旧回测，需要补一层列名适配
